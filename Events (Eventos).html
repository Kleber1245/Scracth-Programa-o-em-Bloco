O que são os Blocos de Eventos?

No Scratch, o botão "Events" (ou "Eventos") é utilizado para definir quando e como os scripts devem começar a ser executados em seus projetos. Eles atuam como gatilhos que iniciam sequências de ações em resposta a interações do usuário ou ocorrências no programa.

Principais Blocos de Eventos:

  • Quando a bandeira verde for clicada: Inicia a execução dos scripts anexados quando o usuário clica na bandeira verde no palco. Este é frequentemente o ponto de partida principal para a maioria dos projetos.
  • Quando a tecla [ ] for pressionada: Inicia a execução dos scripts quando uma tecla específica do teclado é pressionada. Você pode escolher qualquer tecla.
  • Quando este sprite for clicado: Inicia a execução dos scripts quando o sprite em que o bloco está inserido é clicado com o mouse.
  • Quando o cenário mudar para [ ]: Inicia a execução dos scripts quando o fundo do palco muda para um cenário específico.
  • Quando o volume do som > [ ]: Inicia a execução dos scripts quando o volume do som do microfone conectado ao computador ultrapassa um determinado limite.
  • Receber [ ]: Define um gatilho para iniciar os scripts anexados quando uma mensagem específica é recebida de outro sprite ou do palco através dos blocos de envio de mensagens.
  • Enviar [ ]: Envia uma mensagem para todos os sprites e para o palco. Outros scripts que estão esperando para receber essa mensagem (com o bloco "Receber [ ]") serão iniciados simultaneamente.
  • Enviar [ ] e esperar: Similar ao bloco "Enviar [ ]", mas pausa a execução do script atual até que todos os scripts que receberam a mensagem e foram iniciados terminem de ser executados.

Os blocos de eventos são cruciais para tornar seus projetos interativos e dinâmicos, permitindo que diferentes partes do seu código respondam a ações do usuário ou a mudanças no ambiente do projeto.
